## Описание данных

#### Задача: Прогнозирование заключения договора
- **`база.csv`** - данные о бронированиях недвижимости строительной компании

| Признак | Описание |
|---------|----------|
| УИД_Брони | Уникальный идентификатор брони (не используется) |
| ДатаБрони | Дата установления брони |
| ВремяБрони | Время установления брони (GMT+4) |
| ИсточникБрони | Способ оформления: МП / ручная (бинарный) |
| ВременнаяБронь | Подтверждена заявка или нет (бинарный) |
| СледующийСтатус | **Целевой признак**: Продана (1) / Свободна (0) |
| Город | Город проекта |
| ВидПомещения | Тип помещения (только «жилые помещения») |
| Тип | Количество комнат |
| ПродаваемаяПлощадь | Общая площадь помещения |
| Этаж | Этаж расположения |
| СтоимостьНаДатуБрони | Стоимость по прайсу |
| ТипСтоимости | 100% оплата / рассрочка |
| ВариантОплаты | Единовременная / рассрочка |
| ВариантОплатыДоп | Ипотека / Вторичное жилье |
| СкидкаНаКвартиру | Размер скидки (отрицательное = наценка) |
| ФактическаяСтоимостьПомещения | Стоимость сделки |
| СделкаАН | Участие агентства недвижимости (да/нет) |
| ИнвестиционныйПродукт | Инвестиционный договор (да/нет) |
| Привилегия | Договор типа Привилегия (да/нет) |
| Статус лида | Статус клиента из CRM (S/P/F) |

## Задачи

#### Основная часть (6 баллов)

1. Загрузка данных из CSV-файла (кодировка `windows-1251`)
2. Предварительная фильтрация:
   - Оставить только «жилые помещения»
   - Отфильтровать записи с неопределённым статусом
   - Заменить «Продана» → 1, «Свободна» → 0
   - Удалить столбец УИД_Брони
3. Преобразование типов данных:
   - Числовые признаки → числовой тип
   - Бинарные признаки → 0/1
   - Категориальные признаки → one-hot кодирование
   - Обработка поля «Тип» (удаление буквы «к»)
4. Работа с пропущенными данными:
   - «СкидкаНаКвартиру» → 0
   - «Тип» и «ПродаваемаяПлощадь» → медиана
   - Остальные → удаление строк или самостоятельное решение
5. Дополнение данных (Feature Engineering):
   - «Цена за квадратный метр»
   - «Скидка в процентах»
6. Нормализация данных (минимаксная [0;1], скидка → [-0,5; 0,5])
7. Проверка сбалансированности классов
8. Формирование матрицы признаков и целевой переменной
9. Разбиение на обучающую и тестовую выборки (случайный порядок)
10. Обучение модели KNN (KNeighborsClassifier)
11. Обучение модели Decision Tree (DecisionTreeClassifier)
12. Получение прогнозов на train и test выборках
13. Расчёт метрик: Precision, Recall, F-мера
14. Выводы о качестве моделей и интерпретация метрик

#### Дополнительные задания (4 балла)

15. Подбор гиперпараметров:
   - KNN: k от 1 до 40
   - Decision Tree: глубина от 2 до 40
   - Построение графиков зависимости метрик от параметра
16. Обучение логистической регрессии (LogisticRegression), сравнение метрик
17. Обучение SVM (LinearSVC), сравнение метрик
18. Анализ выбросов (boxplot), фильтрация, переобучение модели, сравнение результатов

## Основные функции и библиотеки

В работе используются:
- `pandas.read_csv()` - загрузка CSV с указанием кодировки
- `DataFrame.info()`, `DataFrame.describe()` - информация о данных
- `DataFrame.fillna()` - заполнение пропусков
- `sklearn.preprocessing.LabelEncoder()` - кодирование категориальных признаков
- `sklearn.preprocessing.OneHotEncoder()` - one-hot кодирование
- `sklearn.preprocessing.MinMaxScaler()` - нормализация
- `sklearn.model_selection.train_test_split()` - разбиение выборки
- `sklearn.neighbors.KNeighborsClassifier()` - KNN
- `sklearn.tree.DecisionTreeClassifier()` - дерево решений
- `sklearn.linear_model.LogisticRegression()` - логистическая регрессия
- `sklearn.svm.LinearSVC()` - SVM
- `sklearn.metrics.precision_score()`, `recall_score()`, `f1_score()` - метрики
- `matplotlib.pyplot.boxplot()` - ящик с усами

## Метрики качества

| Метрика | Описание |
|---------|----------|
| Precision (Точность) | Доля истинно положительных среди всех предсказанных положительных |
| Recall (Полнота) | Доля истинно положительных среди всех реальных положительных |
| F-мера (F1-score) | Гармоническое среднее Precision и Recall |
